# ContextBoundary Patterns

Patterns are adoption-oriented documents. They explain how ContextBoundary can be applied around real agent systems without changing the core specification.

Core framework documents define the doctrine. Pattern documents show implementation shapes.

## Contents

- [Runtime Boundary Pattern](runtime-boundary.md) - how ContextBoundary sits between agents and external capability.
- [MCP Boundary Pattern](mcp-boundary.md) - how MCP discovery and invocation are filtered by policy.
- [Local-First Wrapper Pattern](local-first-wrapper.md) - how a drop-in developer experience could wrap common agent tools.

## Boundary

These patterns are not product commitments. They are reference shapes for gateways, proxies, MCP filters, approval services, and audit pipelines that implement ContextBoundary.
